# User Story Map: Multi-level Summary MVP

This user story map outlines the features for the Minimum Viable Product (MVP) of the AI learning tool, focusing on the multi-level summary feature.

## Backbone (User Activities)

| Upload Lecture Notes | Generate Summary | View Summary |
| :--- | :--- | :--- |
| Paste text | Select summary level | Read summary |
| Upload PDF | Initiate generation | Switch between summary levels |


## User Stories

### Release 1: Core MVP

| User Activity | Task | Story |
| :--- | :--- | :--- |
| **Upload Lecture Notes** | **Paste Text** | As a user, I want to paste my lecture notes into a text area so that I can easily provide the content for summarization. |
| | **Upload PDF** | (Deferred) As a user, I want to upload a PDF file of my lecture notes. |
| **Generate Summary** | **Select Summary Level** | As a user, I want to select a difficulty level (e.g., Easy, Medium, Hard) for the summary so that the output matches my learning needs. |
| | **Initiate Generation**| As a user, I want to click a "Generate" button to start the summarization process. |
| **View Summary** | **Read Summary** | As a user, I want to view the generated summary in a clean and readable format. |
| | **Switch Levels** | As a user, I want to easily switch between the different summary levels (Easy, Medium, Hard) to compare them. |
